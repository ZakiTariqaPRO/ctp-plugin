# Gabarit — fichier persona (`personas/<slug>.md`)

Copier cette structure. Remplir uniquement ce qui est connu. Chaque ligne de fait
porte sa source entre crochets : `[interview]`, `[observation]`, `[donnée]`,
`[hypothèse]`, et une date `AAAA-MM-JJ`. Une hypothèse non confirmée reste
étiquetée `[hypothèse]` jusqu'à validation.

```markdown
# Persona — <Nom court>

> Statut : actif · Créé : AAAA-MM-JJ · Dernière mise à jour : AAAA-MM-JJ
> Projet : <nom> · Offre : <ce qu'il achète>

## En une phrase
<Qui + déclencheur + résultat recherché, en une ligne mémorisable.>

## Profil
- Rôle / situation : … [source, date]
- Contexte : … [source, date]
- Décideurs / influenceurs autour de lui : … [source, date]

## Motivations
- Vers (ce qu'il veut) : … [source, date]
- Fuir (ce qu'il évite) : … [source, date]
- Coût de l'inaction : … [source, date]

## 5 Rings of Buying Insights
1. Priority Initiative (déclencheur) : … [source, date]
2. Success Factors (résultats attendus) : … [source, date]
3. Perceived Barriers (freins) : … [source, date]
4. Decision Criteria (critères) : … [source, date]
5. Buyer's Journey (parcours + sources de confiance) : … [source, date]

## Verbatims
> « citation exacte de l'acheteur » — [source, date]

## Implications marketing
- Quoi dire / ne pas dire : …
- Où et quand l'atteindre : …
- Objections à lever en priorité : …

## Hypothèses à valider
- [ ] … [hypothèse, date]

## Journal des changements
- AAAA-MM-JJ — création.
- AAAA-MM-JJ — ajout/modif : … (source).
```

## Règles d'écriture

- Ne jamais écraser un fait en silence. Une contradiction → archiver l'ancien
  dans « Journal des changements » avec sa date, puis mettre le nouveau.
- Mettre à jour « Dernière mise à jour » à chaque modif.
- Garder le doc lisible : préférer des puces courtes aux paragraphes.
- Export HTML optionnel pour présenter à un humain ; la source markdown reste la
  référence vivante.
```
