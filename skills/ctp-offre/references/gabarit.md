# Gabarit — document d'offre

Le skill écrit ce fichier dans le projet sous `offres/<slug>.md`. Un fait = une
source + une date. On n'écrase jamais en silence (mode VALIDATE).

```markdown
# Offre — <Nom de l'offre>

> Dernière mise à jour : <AAAA-MM-JJ>
> Cible : `personas/<slug>.md` (si présent)
> Base de toute page de vente, pricing et communication (skill `ctp-offre`).

## En une phrase
J'aide <cible> à <transformation> grâce à <ta valeur / ton mécanisme>.

## Les 3 P (origine de l'offre)
- **Pain** — problème résolu (cible) : <…> — *(source, date)*
- **Pain vécu** (légitimité) : <…>
- **Passion / hobby** : <…>
- **Pro** (expérience + valeur ajoutée) : <…>

## Transformation & promesse
- **Avant** : <où est la personne avant>
- **Après** : <où elle est après>
- **Ce qu'on promet** : obligation de moyens — <les moyens mis à dispo>
- ⚠️ **Pas de garantie de résultat** (cadre Tariqa PRO).

## Pourquoi toi (irrésistibilité)
- Mécanisme / méthode unique : <…>
- Raison de choisir toi vs alternative / ne rien faire : <…>
- Réducteurs de risque perçu (preuves, cadre, accompagnement) : <…>

## Architecture
| Niveau | Pour qui | Contenu / accès | Rôle |
|---|---|---|---|
| Starter | <…> | <…> | entrée, décider vite |
| Premium | <…> | <…> | cœur de valeur |
| Élite | <…> | <…> | haut + offre d'ancrage |

*(Garder seulement les niveaux qui ont du sens.)*

## Pricing & ancrage (DONNÉE INTERNE — jamais public)
- Fourchette par niveau : <… travail interne>
- Offre d'ancrage : <le niveau haut présenté en premier>
- Justification de la valeur (pourquoi viser au-dessus du budget) : <…>

## Modalités de paiement
- One-shot / échelonné : <choix + pourquoi>
- Logique de responsabilisation : <…>
- Montant dû (pas abonnement) : <oui/non + cadre>

## MVP & go-to-market
- Version simple testable maintenant : <…>
- Chemin le plus rapide vers les premiers clients : <…>

## Conformité
- Dernier audit `ctp-compliant` : <verdict + date>
- Écarts ouverts : <…>

## Hypothèses & à préciser
- ⏳ <points non encore tranchés>

## Journal
- <AAAA-MM-JJ> : <changement / décision> — *(source)*
```

## Règles de remplissage
- Chaque fait : **source** (interview / persona / observation) + **date**.
- Le **pricing reste interne**. Pas de chiffres d'argent dans les sorties publiques.
- Vendre une **transformation**, pas des features.
- Hypothèse non validée → étiquetée comme telle, jamais présentée comme un fait.
- Contradiction avec une version antérieure → archiver l'ancienne en note datée.
