# Gabarit — charte de voix

Écrit dans le projet sous `voix/<slug>.md`. Un fait = source + date. On n'écrase
jamais en silence (mode VALIDATE).

```markdown
# Voix de marque — <Nom du projet>

> Dernière mise à jour : <AAAA-MM-JJ>
> Cible : `personas/<slug>.md` · Positionnement : `positionnement/<slug>.md`
> Charte de tout contenu écrit du projet (skill `ctp-voix`).

## En une phrase
Notre voix est <3-5 adjectifs>, elle parle à <cible> pour lui faire ressentir <émotion>.

## Personnalité & archétype
- Adjectifs de personnalité : <…>
- Archétype (cohérent positionnement) : <Créateur / Magicien / Soignant / …>
- Modèles d'inspiration (et pourquoi) : <…>

## Ton & registre
- Adresse : <tutoiement / vouvoiement> — *(choix du client)*
- Curseurs : chaleureux ↔ pro = <…> · simple ↔ sophistiqué = <…> · calme ↔ énergique = <…>
- Émotion visée : <…>

## Lexique
- **Mots/expressions signature (à utiliser)** : <…>
- **Mots de la cible** (repris d'elle) : <…>
- **À éviter** (corporate, hype, rebute la cible) : <…>
- **Interdits CTP** : hacks · hustle · revenu passif · growth hacking · chiffres d'argent · « Tariqa PRO » seul

## Structures & tics de style
- Structures préférées : <phrases courtes / questions / métaphores / listes / anecdotes>
- Ouverture type (accroche) : <…>
- Clôture / CTA type : <… (pression ? désamorcé ?)>

## Échantillons de référence
- Sonne comme nous : « <extrait> »
- Ne sonne PAS comme nous : « <contre-exemple> »

## Exemples AVANT / APRÈS (ancrage)
1. AVANT : « <phrase banale> » → APRÈS : « <même idée dans la voix> » *(pourquoi)*
2. AVANT : « … » → APRÈS : « … »

## Règles de copy actives (rappel)
Clarté/vulgarisation · phrases courtes · 1 idée/phrase · bénéfice avant
caractéristique · spécifique · hook · 1 seul CTA · éditer (-10-20 %).
Détail : `references/copywriting.md`.

## Hypothèses & à préciser
- ⏳ <points non tranchés>

## Journal
- <AAAA-MM-JJ> : <changement> — *(source)*
```

## Règles de remplissage
- Chaque fait : **source** (interview / persona / positionnement) + **date**.
- Les **échantillons** et **AVANT/APRÈS** sont le cœur — une voix se montre.
- Voix du client ≠ voix CTP : ne jamais coller le style de Zaki par défaut.
- Si le style nuit à la clarté → arbitrer pour la clarté.
- Contradiction avec une version antérieure → archiver l'ancienne en note datée.
