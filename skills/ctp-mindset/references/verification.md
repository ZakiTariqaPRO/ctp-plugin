# Le système de vérification

Le parcours se contrôle lui-même. Coach Zaki coche avec le membre, à voix haute,
à la fin de chaque module. Le membre **voit** sa progression.

---

## Module 0 — Le bilan
- [ ] Les cinq axes sont remplis, chaque réponse passe son test de qualité.
- [ ] L'avantage non-égalitaire est écrit **ou** deux-trois directions sont posées.
- [ ] Le membre a nommé une chose qu'il ramène et qu'il sous-estimait.
- [ ] 3 briques posées.

## Module 1 — L'ancrage
- [ ] Le pourquoi est ancré, et le coût de l'inaction est formulé en opportunités.
- [ ] Les moteurs fragiles ont été testés et renforcés.
- [ ] La situation de départ est décrite (temps, risque supportable, réserve).
- [ ] Les risques de décrochage sont cartographiés et aiguillés vers le bon module.
- [ ] 3 briques posées.

## Module 2 — Toi ↔ Dieu
- [ ] L'intention est écrite et le membre la dit sans hésiter.
- [ ] Les trois moteurs sont classés dans son ordre, avec justification.
- [ ] Le seuil de suffisance est posé.
- [ ] La question-test a servi au moins une fois.
- [ ] 3 briques posées.

## Module 3 — Toi ↔ toi-même
- [ ] La posture dominante est nommée par le membre lui-même.
- [ ] Sa famille de jihad al-nafs est identifiée : dispersé (→ focus) ou
      perfectionniste (→ exécution), et le combat est nommé.
- [ ] Deux ou trois croyances sont identifiées, avec ce qui les démonte.
- [ ] La carte forces / faiblesses existe.
- [ ] Une situation récente a été rejouée en posture responsable.
- [ ] 3 briques posées.

## Module 4 — Toi ↔ ton environnement
- [ ] Le cercle est audité, le membre sait qui il consulte désormais.
- [ ] Le lieu de travail est décidé.
- [ ] Les points d'entrée sont listés et réduits à 3 ou 4.
- [ ] Une habitude est posée en quand / où / quoi.
- [ ] Une checklist existe sur le geste le plus raté.
- [ ] Le membre sait que son comportement est un levier, et il ose facturer son métier.
- [ ] 3 briques posées.

## Module 5 — Le rituel & l'agenda
- [ ] La situation de vie réelle est écrite.
- [ ] La ligne contrainte / budget récupérable est tracée et **acceptée**.
- [ ] Les blocs sont **dans l'agenda**, pas dans une note.
- [ ] La revue du dimanche est posée en récurrent.
- [ ] Trois règles d'hygiène maximum sont choisies.
- [ ] Le membre sait dire ce qu'il coupe.
- [ ] 3 briques posées.

## Module 6 — Le commencement
- [ ] Projet nommé, en une phrase.
- [ ] Engagement daté et écrit.
- [ ] Premier acte public posé, publiquement.
- [ ] Blocs d'agenda tenus **deux semaines**.
- [ ] Les six livrables existent.
- [ ] Validé en call.

---

## Les trois tests transverses

Coach Zaki les applique à n'importe quel moment du parcours.

**Le test de l'action.** Toute action sortie d'un module a-t-elle ses trois
coordonnées — quand, où, quoi exactement ? Sinon ce n'est pas une action.

**Le test des trois briques.** Chaque livrable a-t-il un outil vivant, un
rituel, et un lieu ? Sinon il mourra.

**Le test du renvoi.** Quand le membre a eu besoin du fond, l'a-t-on renvoyé
vers la leçon Circle précise, ou a-t-on réexpliqué à sa place ?
