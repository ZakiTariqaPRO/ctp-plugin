# CTA — qualifier et câbler l'appel à l'action

Le CTA est le cœur de la page : il transforme un visiteur en **rendez-vous** (ou en
contact qualifié). On le **qualifie** avant de le poser.

## 1. Choisir le type

- **Calendrier (recommandé pour service/coaching)** : le visiteur réserve un créneau.
  Outils gratuits : **Calendly** ou **cal.com**. Le membre crée son lien (ex.
  `calendly.com/son-nom/appel-decouverte`).
- **Formulaire** : le visiteur laisse ses infos, le membre rappelle. Outil recommandé
  non-tech : **Tally** (gratuit, no-code). Le membre crée son formulaire → récupère le
  lien (ou le code d'embed).
- **Les deux** : possible, mais **un seul CTA principal** mis en avant (l'autre en secondaire).

## 2. Qualifier (valider avant de poser le bouton)

- **Cohérent avec l'offre ?** Offre premium / accompagnement → privilégier le **call**
  (on ne vend pas un accompagnement par un bouton « acheter »). Petite offre simple →
  un formulaire peut suffire.
- **La destination existe ?** Demander le **lien réel** et vérifier qu'il est valide
  (pas un placeholder). Sans lien, le CTA ne sert à rien.
- **Une seule action.** Ne pas multiplier les boutons concurrents. Le visiteur doit savoir
  exactement quoi faire.
- **Promesse du bouton claire** : « Réserver mon appel découverte » > « Envoyer ». Dire ce
  qui se passe après le clic.

## 3. Câbler dans la page

- **Calendrier** : bouton → lien Calendly/cal.com (ou widget d'embed si le membre veut).
- **Formulaire Tally** : soit un bouton → lien Tally, soit l'**embed** Tally dans la section
  CTA (snippet fourni par Tally). Le plus simple pour démarrer : le **lien**.

## 4. RGPD / données (si formulaire)

- Prévenir le membre : un formulaire collecte des données personnelles. Ajouter une
  **mention de consentement** simple (« En envoyant ce formulaire, j'accepte d'être
  recontacté. ») et préciser **où vont les données** (l'outil, son email).
- Ne pas collecter plus que nécessaire (nom, contact, besoin). Pas de données sensibles.

## 5. Après le clic — le hold-back

Le CTA mène à une **conversation** (call) ou un **contact**, pas à l'achat immédiat. C'est
voulu : la page a donné envie, le call de closing fait le reste. On garde donc volontairement
le détail de l'offre (prix, modalités, contenu complet) **hors de la page**.
